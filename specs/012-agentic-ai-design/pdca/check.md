## Issues Encountered

| Issue | Root cause | Resolution |
|-------|-----------|------------|
| pyright `reportUnusedFunction`(デコレータ登録ツール/validator) | `@agent.tool` / `@agent.output_validator` は「登録のみ」で名前が後続で読まれず dead code 判定 | モジュールレベル関数 + `tools=[...]` 明示リスト、設定値クロージャは「ファクトリが返す」形へ |
| `output_validator` の型が `A | B` union を要求(sentinel `B` は実行時除外) | pydantic-ai がフレームワーク側で `DeferredToolRequests` を事前フィルタするが静的型は知らない | `isinstance` ガード + `# pragma: no cover`(factory docstring に理由明記) |
| `DeferredToolResults.approvals` へ狭い union の dict を渡すと値型不変で型エラー | 外部 API のパラメータ型が `dict[K,V]` 固定 | 呼び出し箇所で `dict(x.items())` 再構築し双方向推論に期待型を見せる |
| ruff `TC002` が pydantic `BaseModel` フィールド注釈を実行時解決と認識しない | ruff 既知の盲点 | `patterns_contracts` 確立済みの `# noqa: TC00x + 理由コメント`抑制をレーン跨ぎで再利用 |
