## Success Pattern OR Mistake Record

### Pattern

- **Problem**: pydantic-ai v2 の停止しうるエージェントは `output_type = [業務出力, DeferredToolRequests(sentinel)]` の union になり、pyright strict 下で ①デコレータ登録ツール/validator の `reportUnusedFunction`、②validator が union 全体署名を強制、③外部 API の `dict` 固定パラメータへの狭い union dict 受け渡しで型エラー、の 3 摩擦を生む。
- **Solution**: ①モジュールレベル関数 + `tools=[...]` 明示(設定値クロージャは「ファクトリが返す」形)、②validator/harness で `isinstance(sentinel)` ガード + `# pragma: no cover`(理由明記)、③呼び出し箇所で `dict(x.items())` 再構築し双方向推論に期待型を見せる。
- **Implementation**: `.sdd/patterns/pydantic-ai-union-output-sentinel-typesafety.md` 参照。ローカル `# type: ignore` ゼロで pyright strict を通す。
- **Benefits**: constitution の lint/type 弱体化禁止を守りつつ設定値注入を型安全化。sentinel の `isinstance` ガードが R6.1「再 defer を業務型に誤型付けしない」を静的に保証。
- **Evidence**: `agent.py:76-160` / `harness.py:162-172`、pyright strict 0 errors、カバレッジ 100%、30 passed。
- Saved to: `.sdd/patterns/pydantic-ai-union-output-sentinel-typesafety.md`
