## Success Pattern OR Mistake Record

### Pattern — Verify-then-strengthen for seeded RED-driver test files

- **Problem**: When TDD test files are authored as a *RED driver* during an
  implementation major (C2 here), the later test-only `(P)` major (C4) risks
  becoming busy-work: the files already exist and pass GREEN. Re-authoring them
  wastes effort; rubber-stamping them leaves *clause-level* coverage gaps invisible
  (a file can pass while silently missing a requirement clause it never asserted).
- **Solution**: Treat each later test-only sub-task as **verify-then-strengthen**,
  not re-author. For every enumerated clause in the task, confirm an existing
  assertion pins it; where a clause is covered only *incidentally* or *implicitly*,
  add the missing assertion. The delta is the value, and it is small and surgical.
- **Implementation**:
  1. Map the task's enumerated clauses to existing assertions in the seeded file.
  2. For each gap, add a targeted assertion that pins the *intent*, not just shape:
     - **Delegation-equivalence** over literal shape: `captured["messages"] ==
       _map_messages(messages)` proves `request()` *routes through* the shared
       helper (Req 11 single-implementation reuse), where a literal-only dict would
       pass even if mapping were re-implemented inline (3.2).
     - **Both span identities** for parity: assert `model_name` *and*
       `provider_name`, not just the provider segment, so a dropped route-stamp
       can't slip through (3.1).
     - **Through-the-transport** coverage of shared helpers: re-pin finish-reason
       mapping via `request()` → `.model_dump()` → `build_response`, not just on the
       helper in isolation, to catch call-site normalisation drift (3.3).
     - **Fail-loud sentinels**: a monkeypatched `acompletion` that detonates on
       reach proves a guard raises *before* the wrapped call (mapping errors stay
       unwrapped, Req 4.3; streaming never silently downgrades, Req 8.2).
  3. Keep the green baseline before the strengthening edit; the new assertion is
     the RED for the gap clause.
- **Benefits**: No duplicated authoring across waves; every requirement clause is
  *explicitly* pinned, not incidentally; the strengthening assertions are exactly
  the regression guards that catch the realistic failure modes (inline drift,
  dropped identity, silent downgrade).
- **Evidence**: do.md tasks 3.1–3.6 — each verified-then-strengthened one clause
  GREEN with a small delta; major 3 closed with `litellm.py` at 100% coverage and
  no test re-authoring.
- Saved to: `.sdd/patterns/tdd-verify-then-strengthen-seeded-tests.md`

### Secondary learning — Honest down-scope of an unprovable assertion

The planned live `num_retries=0` per-request POST-count equality proved *doubly
unprovable* (aiohttp/httpx transport split hides the inference POST; a happy-path
request reveals no retry budget). Rather than assert an unverifiable guarantee, the
assertion was removed, the test renamed to its honest scope, the hermetic
kwarg-passthrough pin retained, and a forced-failure lane documented as the proper
home for the suppression proof. **Principle: a verification you cannot actually run
is worse than a documented gap — it reads as green while proving nothing.**

## Process Improvements

- **Author tasks.md to name the verify-then-strengthen shape up front** when a
  later test major depends on a major that produces RED-driver files — so the
  deliverable is scoped as "verify clauses + close gaps", avoiding both
  re-authoring and rubber-stamping.
- **Flag hermetically-unprovable acceptance criteria at plan time** (the plan *did*
  flag `num_retries=0` as live-only, which is good) — and additionally pre-commit to
  the *mechanism* (forced-failure injection), since "observe one upstream attempt on
  the happy path" was structurally incapable of proving suppression.
- **Reconcile spec wording with shipped reality during the quality-gate major** —
  the BLE001 noqa premise was corrected in code in major 2 but the task text lagged
  until 7.2; carry such "spec-doc amendment" items explicitly across majors.
